# Case Study Craft

A writing skill for turning product design work into portfolio case studies that show judgment, craft, and point of view, not just process boxes.

This repository contains the public version of the **Case Study Craft** skill.

## What this is

Design case studies are often flattened into the same structure:

**Problem -> Research -> Personas -> Journey Map -> Wireframes -> Final Designs**

The result is familiar: everything looks competent, organized, and forgettable.

Case Study Craft was written to push against that default.
It helps structure case studies around:

- decisions
- narrative tension
- tradeoffs
- insights
- outcomes
- the designer's actual voice

Rather than generating generic portfolio filler, the skill is designed to gather real material first and then shape it into a sharper story.

## Core idea

A case study is not primarily about the project.
It is about the designer.

The point is to make visible:

- how you think
- what you notice
- how you decide
- what you care about
- what makes your work distinct

## Recommended GitHub setup

Use a small public repo with a clean name, for example:

- `case-study-craft`
- `design-case-study-skill`
- `case-study-writing-skill`

Put the skill at the root of the repository as `SKILL.md`.
That keeps the URL clean and makes the repo legible immediately. Humans do enjoy making simple things look mysterious, but this one should stay obvious.

## How to publish it

1. Create a new public repository under your GitHub account.
2. Name it something plain and searchable.

5. Commit both files.
6. Optionally pin the repo on your GitHub profile.

## Hosting advice

For this specific thing, **GitHub is the right default**.

Why:

- clean public URL
- version history
- easy updates
- credibility signal for hiring managers and design peers
- no maintenance overhead compared with hosting it yourself

A personal site is better later if you want to turn it into a proper landing page with examples, screenshots, and a stronger editorial frame.
For now, GitHub is enough and probably better.
