# Case Study Craft

A writing skill for turning product design work into portfolio case studies that show judgment, craft, and point of view, not just process boxes.

This repository contains the public version of the **Case Study Craft** skill.

## What this is

Design case studies are often flattened into the same structure:

**Problem -> Research -> Personas -> Journey Map -> Wireframes -> Final Designs**

The result is familiar: everything looks competent, organized, and forgettable.

Case Study Craft was written to push against that default. It helps structure case studies around decisions, narrative tension, tradeoffs, insights, outcomes, and the designer's actual voice.

Rather than generating generic portfolio filler, the skill is designed to gather real material first and then shape it into a sharper story.

## Core idea

A case study is not primarily about the project. It is about the designer.

The point is to make visible:

- how you think
- what you notice
- how you decide
- what you care about
- what makes your work distinct
